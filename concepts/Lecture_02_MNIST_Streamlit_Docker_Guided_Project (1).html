Intro · The Journey

From notebook artefacts to a portable container

We start with two files produced by 01_MNIST_Training.ipynb — mnist_model.keras and mnist_artifacts.pkl — and end with a self-contained Docker image that runs the model behind a drawing UI. Six short steps, all on Windows.

The yellow packet you see flowing left-to-right is your artefact moving through the pipeline — starting as a Python interpreter, accumulating dependencies, getting wrapped in a UI, running locally, getting baked into a Docker image, and finally being served. Each card on the left maps to one stage of this journey. Click "Next" below (or any step in the sidebar) to dive in.

You bring
mnist_model.keras + mnist_artifacts.pkl from the training notebook.
You learn
Embeddable Python, virtualenv, Streamlit drawing canvas, Dockerfile layering, and image serving.
You leave with
A reproducible mnist-streamlit:0.1 image anyone can docker run.
Step 01 · Environment

Install Python 3.12.10 & venv on Windows

We follow the same offline-friendly pattern as 01_setup_environment.bat: download the official embeddable Python zip from python.org, unpack it into a project-local folder, install pip with get-pip.py, and then add virtualenv so we can create an isolated environment. The model was trained on Python 3.12.10 with TensorFlow 2.20.0, so we match those versions exactly.

What you get
A local Python interpreter at mnist-python\python.exe — no admin rights, no system install.
Why embeddable
Zero installer, fully self-contained, easy to delete. Same approach as the BRAC lab setup script.
Why virtualenv
Keeps Streamlit + TensorFlow dependencies pinned per project so you never break another env.

Open Command Prompt (not PowerShell) so the commands below run exactly as written. Create a working folder and step into it. Place mnist_model.keras and mnist_artifacts.pkl here too.

1.2 — Download & extract embeddable Python 3.12.10

Same pattern as the 01_setup_environment.bat file: curl the zip, mkdir a folder for it, tar -xf to extract. Windows 10/11 ships with both curl and tar out of the box.

cmd.exe 
curl https://www.python.org/ftp/python/3.12.10/python-3.12.10-embed-amd64.zip -o python-3.12.10-embed-amd64.zip

mkdir mnist-python
move  python-3.12.10-embed-amd64.zip mnist-python
cd    mnist-python
tar   -xf python-3.12.10-embed-amd64.zip

.\python.exe --version
:: expected output: Python 3.12.10

1.3 — Bootstrap pip and unlock site-packages

The embeddable Python ships without pip and with a ._pth file that disables site imports — both of which break virtualenv. We fix both with two tiny commands.

cmd.exe (still inside mnist-python\) 
:: 1. download the pip bootstrap and install it
curl https://bootstrap.pypa.io/get-pip.py -o get-pip.py
.\python.exe get-pip.py

:: 2. enable `import site` so virtualenv can find packages
echo import site >> python312._pth

:: 3. tidy up the installers
del get-pip.py
del python-3.12.10-embed-amd64.zip

1.4 — Install virtualenv and create the project env

We install virtualenv with the freshly-bootstrapped pip, then create an env called mnist-env in the project root. Activating it puts a clean (mnist-env) prefix on your prompt and pins all future pip install commands to this env.

cmd.exe 
:: still inside C:\mnist-app\mnist-python — install virtualenv
.\Scripts\pip.exe install virtualenv

:: hop back to the project root and create the env there
cd ..
mnist-python\Scripts\virtualenv.exe -p mnist-python\python.exe mnist-env

:: activate it — your prompt should now start with (mnist-env)
mnist-env\Scripts\activate

:: sanity check
python --version      :: -> Python 3.12.10
where python           :: -> ...\mnist-env\Scripts\python.exe
Checkpoint — your prompt now reads (mnist-env) C:\mnist-app> and python --version prints Python 3.12.10. You have a fully isolated, project-local interpreter ready for TensorFlow.
Step 02 · Dependencies

Create requirements.txt and install

We pin every package to the exact version the model was trained against (see the table in the training notebook). Pinning makes the env reproducible and — crucially — guarantees that the .keras file loads with no version-skew warnings. We add Streamlit and the drawable canvas component on top so we can draw digits in the browser.

2.2 — Install everything into the active env

Make sure your prompt still says (mnist-env). Upgrade pip first (TensorFlow wheels are large and modern pip resolves faster), then install from the file. The first run may take a few minutes — TensorFlow alone is ~ 600 MB.

cmd.exe (inside C:\mnist-app, with mnist-env active) 
python -m pip install --upgrade pip

:: confirm the critical packages landed at the right versions
python -c "import tensorflow as tf, streamlit as st; print(tf.__version__, st.__version__)"
:: expected output: 2.20.0 1.38.0
If you see ERROR: Could not find a version for TensorFlow — double-check that python --version says 3.12.10 and that you are on 64-bit Windows. TensorFlow 2.20 has no 32-bit wheel.
Step 03 · Webapp

Build the Streamlit drawing app

The training notebook saved a tiny Flatten → Dense(256) → Dropout → Dense(128) → Dropout → Dense(10, softmax) network as mnist_model.keras, plus a metadata pickle mnist_artifacts.pkl with the preprocess function, class labels, and the expected input shape (28, 28). The app below:

  • Loads the .keras model and the pickle once via @st.cache_resource.
  • Renders a 280×280 black canvas where the user draws a digit in white (MNIST style).
  • Down-samples that to 28×28 grayscale, divides by 255 (the saved preprocess), and runs model.predict.
  • Shows the predicted digit, its confidence, and a bar chart of all class probabilities.

3.1 — File layout

After this step your project folder should look like this:

C:\mnist-app\ 
C:\mnist-app\
├── mnist-python\            # embeddable Python (Step 1)
├── mnist-env\               # virtualenv (Step 1)
├── requirements.txt         # pinned deps (Step 2)
├── mnist_model.keras        # trained Keras model
├── mnist_artifacts.pkl      # labels + preprocess + metadata
└── app.py                   # <-- we create this next

3.2 — Create app.py

Save the following as C:\mnist-app\app.py. Every section is commented so you can map it back to the architecture.

app.py 
# ============================================================
# MNIST Digit Recognizer — Streamlit front-end for mnist_model.keras
# ============================================================
from pathlib import Path
import pickle

import numpy as np
import streamlit as st
import tensorflow as tf
from PIL import Image
from streamlit_drawable_canvas import st_canvas

# ---- IMPORTANT — re-declare `normalize` exactly as it was in the training notebook.
# mnist_artifacts.pkl stores artifacts["preprocess"] = normalize as a *function reference*
# (pickle records the module + name, not the body). When pickle.load() runs here, Python
# tries to resolve __main__.normalize — so this top-level definition must exist BEFORE
# load_artifacts() is called, otherwise you will hit:
#     AttributeError: Can't get attribute 'normalize' on <module '__main__' from 'app.py'>
def normalize(images: np.ndarray) -> np.ndarray:
    return images.astype("float32") / 255.0

# ---- paths (relative to this file so it works in Docker too) ----
ROOT          = Path(__file__).parent
MODEL_PATH    = ROOT / "mnist_model.keras"
ARTIFACT_PATH = ROOT / "mnist_artifacts.pkl"

# ---- page config ----
st.set_page_config(
    page_title="MNIST Digit Recognizer",
    page_icon="📝",
    layout="centered",
)

# ---- cached loader: runs once per server, not once per click ----
@st.cache_resource(show_spinner="Loading MNIST model…")
def load_artifacts():
    with open(ARTIFACT_PATH, "rb") as f:
        artifacts = pickle.load(f)
    model = tf.keras.models.load_model(MODEL_PATH)
    return artifacts, model

artifacts, model = load_artifacts()
class_labels = artifacts["class_labels"]            # [0,1,…,9]
input_shape  = tuple(artifacts["input_shape"])      # (28, 28)

# ---- preprocess: rgba canvas -> (1, 28, 28) float32 in [0, 1] ----
def preprocess(rgba: np.ndarray) -> np.ndarray:
    img  = Image.fromarray(rgba.astype("uint8")).convert("L")
    img  = img.resize((input_shape[1], input_shape[0]), Image.LANCZOS)
    arr  = np.asarray(img, dtype="float32") / 255.0
    return arr[np.newaxis, ...]

# ============================== UI ==============================
st.title("📝  MNIST Digit Recognizer")
st.caption(
    "Draw a digit (0–9) in the black box. "
    "The model expects a thick white stroke on a black background — "
    "the same colour scheme it was trained on."
)

col_draw, col_pred = st.columns([1.05, 1])

with col_draw:
    st.subheader("1. Draw")
    canvas = st_canvas(
        fill_color="rgba(0,0,0,0)",
        stroke_width=18,
        stroke_color="#FFFFFF",
        background_color="#000000",
        height=280,
        width=280,
        drawing_mode="freedraw",
        key="mnist-canvas",
    )
    predict_clicked = st.button("🚀  Predict", type="primary", use_container_width=True)

with col_pred:
    st.subheader("2. Prediction")
    if predict_clicked and canvas.image_data is not None:
        x = preprocess(canvas.image_data)

        if x.sum() < 1e-3:
            st.warning("The canvas looks empty — draw a digit and click Predict.")
        else:
            probs = model.predict(x, verbose=0)[0]
            top   = int(np.argmax(probs))

            st.metric(label="Predicted digit",
                      value=str(class_labels[top]),
                      delta=f"{probs[top]*100:.1f}% confident")
            st.progress(float(probs[top]))

            st.caption("Full softmax distribution")
            st.bar_chart(
                {str(c): float(p) for c, p in zip(class_labels, probs)}
            )
    else:
        st.info("Draw something on the left, then press **Predict**.")

# ---- sidebar diagnostics ----
with st.sidebar:
    st.header("Model card")
    st.write("**Architecture:** Flatten → Dense(256, ReLU) → Dropout → Dense(128, ReLU) → Dropout → Dense(10, softmax)")
    st.write(f"**TensorFlow:** `{artifacts.get('tensorflow_version', tf.__version__)}`")
    test_metrics = artifacts.get("test_metrics", {})
    if test_metrics:
        st.write(f"**Test accuracy:** `{test_metrics.get('accuracy', 0):.4f}`")
        st.write(f"**Test loss:**     `{test_metrics.get('loss', 0):.4f}`")
Three implementation notes —
• Why the normalize function is re-declared at the top of app.py: the training notebook saved artifacts["preprocess"] = normalize, and pickle records that as the reference __main__.normalize. When Streamlit unpickles the file, Python searches the current __main__ (i.e. app.py) for a function with that exact name. If it is missing you will see AttributeError: Can't get attribute 'normalize' on <module '__main__' from '…/app.py'>. The body must be byte-identical to the notebook's version (images.astype("float32") / 255.0).
• The training pipeline normalised pixels with x / 255.0 on a black-background image. Our canvas already produces white strokes on black, so we replicate the exact same scaling — no inversion needed.
• @st.cache_resource keeps the loaded Keras model in memory across reruns; without it Streamlit would reload the model on every keystroke.

4.1 — Smoke test

  1. Draw a clearly recognisable 3 in the black box.
  2. Click Predict.
  3. Confirm the right pane shows Predicted digit: 3 with high confidence (typically > 95 %).
  4. Try a few more digits — especially 4 vs 9 and 1 vs 7 — to see how confident the model is on tricky pairs.
Checkpoint — if the app loads and returns the correct digit for a handful of examples, your local pipeline is complete. The next steps wrap it into a portable container.

4.2 — Common error you may hit

If the page crashes with AttributeError: Can't get attribute 'normalize' on <module '__main__' from '…/app.py'> — it means the normalize function that pickle stored inside mnist_artifacts.pkl cannot be resolved in your current app.py. Make sure the following block (copied verbatim from the training notebook) sits at the top of app.py, before load_artifacts() is called: 
def normalize(images: np.ndarray) -> np.ndarray:
    return images.astype("float32") / 255.0
Save the file and refresh the browser — Streamlit will reload app.py and the artifacts will unpickle cleanly.
Step 05 · Image

Create the Docker image

A container is just a filesystem snapshot with a recipe for which command to run. We base ours on python:3.12.10-slim (a 70 MB Debian-slim image that already has Python 3.12.10 pre-installed — same version we developed against), install the same requirements.txt, then copy app.py and both model artefacts in.

Prerequisite: install Docker Desktop for Windows and make sure the whale icon in your system tray is green ("Docker Desktop is running"). Verify with docker --version in a fresh Command Prompt.

5.1 — Create the Dockerfile

Save this exact filename (Dockerfile, no extension) in C:\mnist-app:

Dockerfile 
# ---- base image: matches the Python version used in training ----
FROM python:3.12.10-slim

# keep python output unbuffered + don't write .pyc files inside the image
ENV PYTHONDONTWRITEBYTECODE=1 \
    PYTHONUNBUFFERED=1 \
    PIP_NO_CACHE_DIR=1

WORKDIR /app

# ---- system deps for Pillow / streamlit-drawable-canvas ----
RUN apt-get update && apt-get install -y --no-install-recommends \
        curl \
        libglib2.0-0 \
    && rm -rf /var/lib/apt/lists/*

# ---- python deps (cached layer: copy file first, then install) ----
COPY requirements.txt ./
RUN pip install --upgrade pip && pip install -r requirements.txt

# ---- copy application + model artefacts ----
COPY app.py mnist_model.keras mnist_artifacts.pkl ./

# ---- runtime config ----
EXPOSE 8501
HEALTHCHECK --interval=30s --timeout=10s --start-period=20s \
  CMD curl --fail http://localhost:8501/_stcore/health || exit 1

ENTRYPOINT ["streamlit", "run", "app.py", \
            "--server.port=8501", \
            "--server.address=0.0.0.0", \
            "--browser.gatherUsageStats=false"]

5.2 — Add a .dockerignore

Without this, Docker would copy your entire mnist-python\, mnist-env\ and any cache directories into the build context — easily a 2 GB transfer that you don't want in the image. The file below trims it down to just the four things the image actually needs.

.dockerignore 
# local virtualenv & embeddable python — recreated inside the image
mnist-env/
mnist-python/
*.zip

# python & tooling caches
__pycache__/
*.pyc
.pytest_cache/
.ipynb_checkpoints/

# IDE / OS noise
.vscode/
.idea/
Thumbs.db
.DS_Store

# docs & misc artefacts that aren't needed at runtime
*.md
*.html
*.bat
*.ipynb
Why the layer order matters — we COPY requirements.txt before the rest of the code, then run pip install. Docker caches that whole layer. Next time you edit app.py but leave dependencies untouched, the slow pip install step is skipped entirely and the rebuild takes seconds instead of minutes.
Step 06 · Serve

Containerise & serve

Everything is in place. Build the image, run it, and open http://localhost:8501 in any browser on your machine. From this point on, nothing about your host matters — anyone with Docker can run this exact same container and get an identical app.

6.1 — Build the image

cmd.exe (inside C:\mnist-app) 
docker build -t mnist-streamlit:0.1 .

:: first build takes ~2-4 minutes (pip downloads + tensorflow wheel)
:: subsequent builds use the layer cache and take ~10 seconds

docker images mnist-streamlit
:: REPOSITORY        TAG    IMAGE ID       CREATED          SIZE
:: mnist-streamlit   0.1    9a4b3...       1 minute ago     1.8GB

6.2 — Run the container

The -p 8501:8501 flag maps the container's port to your host. The --name gives it a friendly handle so we can stop / restart it easily. --rm tells Docker to delete the container (not the image) when it exits, keeping things tidy.

cmd.exe 
docker run --rm -p 8501:8501 --name mnist-app mnist-streamlit:0.1

:: ----- expected output -----
::  You can now view your Streamlit app in your browser.
::    URL: http://0.0.0.0:8501

:: open in your browser:
start http://localhost:8501

6.3 — Useful day-2 commands

cmd.exe (cheat-sheet) 
:: run in the background so you get your prompt back
docker run -d -p 8501:8501 --name mnist-app mnist-streamlit:0.1

:: tail the live logs
docker logs -f mnist-app

:: stop & remove the running container
docker stop mnist-app

docker tag  mnist-streamlit:0.1 ghcr.io/<your-handle>/mnist-streamlit:0.1
docker push ghcr.io/<your-handle>/mnist-streamlit:0.1
Port mapping
-p 8501:8501 ⇒ host port (left) maps to container port (right). Pick any free host port if 8501 is taken.
Image size
~1.8 GB — TensorFlow dominates. For a leaner image swap to TF-Lite or use a multi-stage build.
Reproducibility
Every dependency is pinned. The image you build today will produce the same predictions in 2027.
